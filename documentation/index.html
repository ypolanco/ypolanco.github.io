Introduction and Features

BRAX is a modern responsive personal portfolio template built with HTML5, CSS, Bootstrap and jQuery.

  • Responsive on mobile, tablet and desktop.
  • Hero area with typed effect (optional).
  • AJAX contact form.
  • Parallax effect.
  • Google Fonts and Font Awesome.

In this documentation we will cover the necessary steps to use this template effectively.

Installation

  1. Download the latest version of the template and extract the directory "brax-html-template" anywhere on your harddrive.
  2. Follow the next few sections for the template modification and customization to fully adapt the template according to your needs.
  3. When you are finished with the customization, upload the modified contents of the "brax-html-template" directory to your Internet hosting provider (usually by FTP).

For more detailed instructions about transferring the template to your web server, please read the manual from your Internet hosting provider.

The Bootstrap Grid

The template uses Twitter Bootstrap framework (version 4) for its numerous advantages. One of the main advantage is organizing the content in a 12 columns grid. It is fully responsive.

The grid is divided into rows which are divided into columns. Here is an example from the official Bootstrap site:

In the example above, there are four rows in the grid as it can be seen according to the image and the code below. On a medium sized device, we can have together 12 elements of columns with CSS class col-md-1 in one row. Or we can have two equal sized columns taking each a half of the row if we mark the column classes col-md-6, etc. The desired layout can be achieved by organizing the content into such columns and rows.

For more information about Bootstrap, please visit http://getbootstrap.com/.

Template Customization

Configuring the HTML file

The template has just one HTML file to modify: index.html. Feel free to modify it according to your needs. At least you need to change the texts -- this is straightforward, just replace any text with the text you desire.

Note: Basic knowledge of HTML is an advantage.

The title of the page and the meta tags author, description and keywords can be found in the page header.

The page has a menu at the top, in the HTML it starts with the comment <!-- HEADER -->. You can rearrange the menu as you want. A menu item has a code like this: 

	<a class="nav-link" href="#about">About</a>
You can change it easily to point to a different part of the page, for example: 
	<a class="nav-link" href="#testimonials">Testimonials</a>

To configure the social icons, find the occurences of this types of links: 

	<a href="#"><i class="fa fa-facebook"></i></a>
Replace the symbol # with the social page URL.

The Services are configurable under the code which starts with <!-- SERVICES -->. A service block looks like this: 

	<div class="box">
		<div class="circle"><i class="fa fa-cogs"></i></div>
		<h3>Coding</h3>
		<p>Praesent magna neque, dapibus sit amet eros sed, aliquam interdum felis. Nulla rhoncus ipsum dui.</p>
	</div>
In the h3 tag there is the service title and below is the service text. The service image is in the form of Font Awesome text - like this: <i class="fa fa-cogs"></i>. You can use any of the Font Awesome icon -- see http://fontawesome.io/icons/. (OPTIONAL) To add a nice animation effect on the service boxes, replace the first line with: 
	<div class="box animation" data-animation="bounceInUp">

The Portfolio is configurable under the code which starts with <!-- PORTFOLIO -->. Each portfolio item has the form of: 

	<div class="item col-md-6 col-lg-4 photography" data-category="photography">
		<a title="Custom description with a <a href='#'>URL</a>. Lorem ipsum dolor sit amet, consectetur adipiscing elit." data-rel="prettyPhoto"
		   href="img/portfolio/portfolio1.jpg">
			<div class="item-wrap">
				<img class="img-fluid" src="img/portfolio/portfolio1.jpg" alt="Photography">
			</div>
			<span class="magnifier"></span>
		</a>
	</div>
It's important that the data-category (for example "photography") is also present in the class atribute of the most outer div. We can than create a filter item: 
	<a class="filter" data-filter=".photography">photography</a>
These filter items are in the html code above the group of portfolio items. Feel free to adjust the texts in the title attribute, the image src attribute and the filters as you want.

The Testimonials are configurable under the code which starts with <!-- TESTIMONIALS -->. A testimonial block is of the form: 

	<div class="gallery-cell">
		<div class="testimonial">
			<img class="testimonial-avatar"
				 src="img/client1.jpg">
			<q class="testimonial-quote">Lorem ipsum dolor sit amet, consectetur adipiscing elit. Duis
				mauris
				ex, gravida ut leo eu, rhoncus porta orci. Fusce vitae rutrum nulla.</q>
			<span class="testimonial-author">Joe Smith, CEO</span>
		</div>
	</div>
Again, change the texts as needed.

Configuring the javascript options

There are three javascript options you can configure. These options are located in the file js/config.js

/* Edit the array as you wish. It will contain the typed strings. */
var config_typedArray = ["I am web developer.", "I am creative."];

/* Disable Parallax effect on touch devices for performance reasons. */
var config_disableParalaxOnTouchDevices = true;

/* Enable typed effect  */
var config_typed = true;

/* Google map parameters */
var config_mapLat = 51.509036; // latitude
var config_mapLng = -0.123280; // longitude
var config_mapZoom = 16; // map zoom
var config_mapTitle = 'My company' // marker title

Configuring Google Maps

To change the address of the google maps widget, you first need to know the latitude and longitude of the place you want to display. To find these coordinates, just go to maps.google.com and click on any point. A box bellow will appear with the coordinates. You then need to edit the file config.js, where you can change these coordinates (config_mapLat, config_mapLng).

Configuring the contact form

To use the contact form, you need to have PHP support on your hosting. You also need to change the first line in the file api/contact.php: 

$configEmail = "hello@example.com";
These is the e-mail of the recipient, i.e. the person who recieves the e-mails send via the contact form. Adjust it according to your needs.
